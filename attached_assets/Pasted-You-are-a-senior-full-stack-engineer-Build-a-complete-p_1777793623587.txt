==================================================
BACKEND APIs TO BUILD
==================================================

AUTH ROUTER (/auth):
POST /auth/register — register user or vendor (role in body), auto-create vendor profile if role=vendor
POST /auth/login — login, return JWT token + user data

VENDORS ROUTER (/vendors):
GET /vendors — list with filters: category, area, city, min_price, max_price, min_rating, capacity, search, featured, page, limit. Return: {total, page, pages, vendors[]}
GET /vendors/me/profile — get logged-in vendor's profile [vendor only]
GET /vendors/{id} — get vendor detail + increment profile_views
POST /vendors — create vendor listing [vendor only]
PUT /vendors/{id} — update vendor listing [owner only]
POST /vendors/{id}/images — upload image via Cloudinary multipart [owner only]
DELETE /vendors/{id}/images/{image_id} — delete image [owner only]
POST /vendors/{id}/packages — add pricing package [owner only]
PUT /vendors/{id}/packages/{pkg_id} — update package [owner only]
DELETE /vendors/{id}/packages/{pkg_id} — delete package [owner only]

USERS ROUTER (/users):
GET /users/me — get current user profile
PUT /users/me — update name, phone, avatar_url
POST /users/favorites/{vendor_id} — toggle favorite (add if not exists, remove if exists). Return {favorited: bool}
GET /users/favorites — get all favorited vendors
POST /users/bookings — create booking request {vendor_id, package_id, event_date, guest_count, notes}
GET /users/bookings — get user's bookings
GET /users/reviews/{vendor_id} — get reviews for vendor (public)
POST /users/reviews/{vendor_id} — add review {rating, comment} [user only, once per vendor]
  — also auto-calculate and update vendor.rating and vendor.total_reviews

BOOKINGS ROUTER (/bookings):
GET /bookings/vendor — get all bookings for logged-in vendor [vendor only]
PUT /bookings/{id}/status?status=approved|rejected|cancelled — update status [vendor only]

ADMIN ROUTER (/admin):
GET /admin/stats — return {total_users, total_vendors, verified_vendors, pending_vendors, total_bookings, total_reviews}
GET /admin/vendors — list all vendors
PUT /admin/vendors/{id}/approve — set verified=True
PUT /admin/vendors/{id}/suspend — toggle is_suspended
PUT /admin/vendors/{id}/feature — toggle is_featured
DELETE /admin/vendors/{id} — delete vendor + cascade
GET /admin/users — list all users
PUT /admin/users/{id}/block — toggle is_blocked

AUTH SYSTEM
==================================================

- JWT tokens with HS256, expiry from env
- bcrypt password hashing
- 3 dependency functions: get_current_user, require_vendor (role=vendor or admin), require_admin (role=admin)
- Block check: blocked users get 401
- Suspended vendors still authenticate but their listings hidden from public

==================================================
IMPORTANT IMPLEMENTATION NOTES
==================================================

1. Tables auto-create on startup via Base.metadata.create_all(bind=engine)
2. CORS: allow origins from FRONTEND_URL env + localhost:5173
3. Password hashing: always bcrypt, never store plaintext
4. JWT: include user id (as string "sub") and role in payload
5. Rating calculation: when new review added, recalculate average of ALL reviews for that vendor
6. Vendor profile_views: increment by 1 on each GET /vendors/{id}
7. Image upload: use multipart/form-data, upload to Cloudinary, store secure_url + public_id
8. Cover image: set vendor.cover_image_url to first uploaded image if not set
9. Features column in packages: store as JSON string e.g. '["Full day coverage", "2 photographers"]'
10. Favorite toggle: if exists delete it, if not create it. Return {favorited: bool}
11. Frontend API service: axios interceptor auto-attaches Bearer token from localStorage
12. On 401 response: clear localStorage and redirect to /login
13. Admin promotion: do via SQL: UPDATE users SET role='admin' WHERE email='...'
14. All money values in PKR (Pakistani Rupees)
15. Default city: Karachi
